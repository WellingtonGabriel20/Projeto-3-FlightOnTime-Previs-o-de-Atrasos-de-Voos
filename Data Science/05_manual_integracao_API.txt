Guia de Integração do Back-End com o Modelo de Previsão de Atrasos
=====================================================================

*Resumo da integração:

A API recebe dados brutos, aplica exatamente as mesmas transformações do treino usando artefatos já salvos no arquivo ".pkl", executa o modelo e retorna a previsão.

------------------

*Contrato de integração (API)

A API deve receber apenas dados conhecidos antes do voo:

Input:
{
  "icao_empresa": "TAM",
  "icao_aerodromo_origem": "SBGR",
  "icao_aerodromo_destino": "SBRJ",
  "partida_prevista": "2026-02-10T14:30:00",
  "tempo_voo_estimado_hr": 1.2,
  "distancia_km": 350.3
}
Colunas que o modelo recebe e seus respectivos tipos:
{
    "icao_empresa",                <--->   #object
    "icao_aerodromo_origem",       <--->   #object
    "icao_aerodromo_destino",      <--->   #object
    "turno"                        <--->   #object
    "tempo_voo_estimado_hr",       <--->   #float
    "mes",                         <--->   #int
    "distancia_km"                 <--->   #float
}
Output e seus respectivos tipos:
{
  "previsao_atraso": 1,            <--->   #int
  "probabilidade_atraso": 0.89     <--->   #float
}

threshold = 0.45
Um voo é considerado como atrasado se a probabilidade for maior ou igual a 0.45

Portanto, sugerimos que a informação que aparece para o usuário seja algo do tipo:
---------------
Caso ATRASADO:

"voo_atrasado" = 1   --->    "Atrasado"
"probabilidade_atraso" = 0.45   --->   "45% de chance de ser atrasado"
---------------
Caso PONTUAL:

"voo_atrasado" = 0   --->    "Pontual"
"probabilidade_atraso" = 0.37   --->   "63% de chance de ser pontual"
---------------
